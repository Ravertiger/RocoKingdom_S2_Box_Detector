# Roco Box Detector 使用说明

## 快速开始

1. 解压后双击 `RocoBoxDetector.exe`
2. 屏幕变暗，用鼠标拖拽框选战斗时惊喜盒子出现的区域，可以先进入一场战斗再确认框选位置，按 `Enter` 确认
3. 检测自动开始，屏幕出现悬浮窗

## 重要：建议使用截图模式

## 悬浮窗按钮

| 按钮 | 功能 |
|------|------|
| `⬤` / `○` | 调试截图开关。平常关着，需要看匹配效果时打开 |
| `🔍` / `◌` | 预览窗口开关。显示蓝框/黄框/绿框位置，调试用 |
| `▣` / `□` | 画面覆盖框开关。把检测框直接画在游戏上 |
| `↻` | 重新框选游戏区域 |
| `⚙` | 打开设置面板 |
| `✕` | 退出程序 |
| `清空` | 删除记录 |

- 拖动标题栏可以移动位置
- 右下角拖拽可以调整大小
- 底栏统计次数

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Shift+R` | 重新框选 ROI |
| `Ctrl+Shift+Q` | 退出程序 |
| `Ctrl+Shift+H` | 显示/隐藏面板 |

## 设置面板

`⚙` 打开，各标签页功能：

| 标签页 | 改什么 |
|--------|--------|
| Anchor | 蓝框（盒子整体）的匹配参数和模板列表 |
| 黄框1 | 第一个图案区域的位置和大小 |
| 黄框2 | 第二个图案区域的位置和大小 |
| 样本1 | 第一个图案的模板和匹配参数 |
| 样本2 | 第二个图案的模板和匹配参数 |
| Runtime | 帧率、分辨率预设、跳帧间隔等 |
| Debug | 预览窗口、调试截图开关 |
| 悬浮窗 | 顶部状态栏样式 |
| 提示文字 | 识别历史面板的样式和颜色 |

**改完记得点"保存并应用"。**

## 注意事项

- **游戏分辨率变更**：调整游戏分辨率（窗口大小）后，画面比例发生变化，**必须重新框选 ROI**，否则蓝框可能识别不到。框选后去 Runtime 确认分辨率预设匹配当前游戏窗口。
- **设置面板改完后务必点"保存并应用"**，否则重启后配置丢失。

## 常见问题

**Q: 蓝框一直识别不到？**
- 检查 Runtime → 分辨率预设是否选对
- 调整 Anchor 标签页的"匹配阈值"（降低到 0.55-0.60）
- 调整"粗筛阈值"（降低或设为 0 关闭）

**Q: 蓝框识别到了但绿框匹配不准？**
- 降低样本1/样本2 的"匹配阈值"
- 调整黄框1/黄框2 的位置使其对准目标图案
- 增加模板样本的数量和多样性

**Q: 游戏全屏时悬浮窗看不见？**
- 游戏设为无边框窗口模式即可

**Q: 预览感觉卡顿？**
- Runtime → Anchor跳帧间隔 调大（如 2 或 3）
- 关掉预览窗口（🔍）
- 关掉画面覆盖框（▣）
- 关掉调试截图（⬤）

**Q: 想要识别更多图案？**
- 样本1 / 样本2 标签页 → `+ 新增图案` → 添加模板图片 → 保存并应用
- 黄框2 默认关闭，需要先在黄框2 标签页勾选"启用区域二"

**Q: 程序打不开？**
- 不要放在中文路径下（虽然已尽量兼容，但仍可能有坑）
- 管理员权限运行试试

## 功能亮点

- **双区域独立识别** — 同时识别盒子中下层的两个属性标志（如属性 + 种族），结果以 `普通 + 物防` 格式展示
- **多帧采样投票** — 蓝框锁定后截取多帧子图，投票决定最终结果，2 票即提前终止，抗干扰能力强
- **自适应缩放** — 模板搜索范围根据框选区域自动缩放，不同分辨率下都能准确定位
- **画面覆盖框** — 检测框（蓝/黄/绿）直接覆盖在游戏画面上，方便实时观察匹配效果
- **识别历史面板** — 半透明悬浮面板，显示历史识别记录，每种图案独立颜色，支持拖动和缩放
- **可视设置面板** — 无需手动编辑配置文件，所有参数（阈值、缩放、模板、Sub-ROI 位置等）可视化调整
- **高斯权重蒙版** — Pattern 模板匹配时中心区域权重更高，边框背景的干扰被大幅压低
- **极速截屏** — 采样阶段直接截取子区域，单帧截屏 < 5ms，严格按设定间隔执行

## 工作原理

```
框选 ROI → anchor(蓝框) 匹配 → 命中锁定 → 延迟 0.3s → 连续截取 5 帧子区域
→ 分批匹配 pattern(绿框) → 多帧投票 → 2 票通过即输出结果
```

Anchor 命中后锁定位置，采样期间不重新检测蓝框，截屏和匹配解耦——截屏按精确间隔执行，匹配延迟到采样完成后一次性运行。

## 模板准备

### Anchor 模板（惊喜盒子整体——建议选取旋转完成后三个图标发光的几帧）
位置：`templates/box_anchor/`

- 包含三层盒子主体、顶部结构、左右边缘
- 不要包含太多草地、天空、粒子特效
- 建议 10~12 张覆盖主要光照/角度变化

### Pattern 1 模板（精灵类型中层图案）
位置：`templates/patterns/`（区域一）

- 只包含中层侧面的目标图标
- 含少量周围边框，图案特征完整
- 不同光照/颜色变化各一张

### Pattern 2 模板（精灵加成下层图案）
位置：`templates/patterns_2/`（区域二）

- 只包含下层侧面的目标图标
- 含少量周围边框，图案特征完整
- 不同光照/颜色变化各一张

## 性能

- 截屏阶段单帧 < 5ms，5 帧总截屏 < 0.1s
- 匹配阶段每帧 ~0.2s，提前投票可跳过后续帧
- 性能瓶颈在 pattern 匹配（模板数 × scale 档位），与截屏解耦后不影响采样精度
- 关预览窗口可节省少量 CPU，关画面覆盖框不影响性能（仅绘制开销）

## 调参优先级

1. 确认蓝框命中 → 调整 anchor threshold 和模板质量
2. 确认黄框对准目标 → 调整 sub_roi / sub_roi_2 的四个 ratio
3. 提高投票通过率 → 降低 min_average_score、增加样本多样性
4. 提速 → 减少 scale_steps、降低 max_frames
